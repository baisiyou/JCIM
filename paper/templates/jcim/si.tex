\subsection{Episode Construction}

The main evaluation contains 200 fixed-start episodes generated by three-step \emph{replace-only} random walks in the test-side edit graph (add/remove edges are indexed but excluded from walks and from the primary planner). We require initial normalized distance at least 1.0 and apply target jitter with per-property amplitudes $(0.02,0.15,7.5)$, strictly below evaluation tolerances $(0.03,0.20,10.0)$, so the generating endpoint remains a valid solution. Starts appearing as either endpoint of a training pair are rejected. Every baseline receives identical start and target values.

\paragraph{Two search protocols.}
\textbf{Observed-graph search} reads precomputed RDKit labels from DuckDB for every expanded neighbor; this is the zero-query primary baseline and reaches 100\% on endpoint-walk episodes. \textbf{Hidden-label search} withholds cached labels during planning and increments a query counter the first time each canonical SMILES $m$ reveals $p(m)$ within an episode (unique-node accounting; episode start free). Live RDKit recomputation is redundant with the index and is used only as a protocol device. Query counts therefore measure label-access budgets, not docking/DFT/experimental cost.

\section{Self-Built Supervised Editor (Not Official M$^4$olGen)}
\label{app:supervised_proxy}

Official M$^4$olGen GRPO weights and an executable Stage~II pipeline were unavailable. We therefore trained a \emph{self-built supervised editor}: a target-conditioned MLP that scores observed replace edges by predicted distance improvement. The \emph{greedy} variant accepts the top policy edge after one oracle check per step ($\approx$3.8 queries; 18.5\% success). The \emph{matched-probes} variant reveals the top-5 policy edges per step and accepts the best true-distance candidate ($\approx$12.3 queries; 38.5\% success) on the same 200 endpoint-walk episodes as Table~\ref{tab:fair}. Neither variant is a reproduction of M$^4$olGen's multi-agent reasoner or GRPO optimizer, and query budgets remain below EditGraph's 29 even in the matched-probe setting. We keep these numbers out of the abstract and do not treat them as peer baselines for main scientific claims.

\section{Prompt-Only LLM Strawman}
\label{app:llm_strawman}

A small local prompt-only LLM (no graph adjacency, no fragment tools, no property calculator in the loop) achieves 0\% success on the same 200 endpoint-walk episodes (QED/LogP/MW MAE $0.100$/$0.949$/$32.4$; $\approx$0.9 queries). This paper is not an LLM-agent study; the comparison has negligible information value for EditGraph and is retained only to document that unconstrained language-model editing without search tools is not a serious baseline here. Stronger tool-augmented or proprietary agents are out of scope.

\section{GraphGA Fairness Experiments}
\label{app:graphga_fairness}

Code: \texttt{src/in\_graph\_ga.py}, \texttt{scripts/18\_fair\_baseline\_comparison.py}; results in \texttt{results/fair\_baseline\_comparison/}. In-graph GA mutates only along observed replace-only edges from the DuckDB index (identical action space to EditGraph). De~novo GraphGA uses official mol-opt crossover/mutation and may invent non-library molecules (1\% in-library at budget 200). On endpoint-walk targets reverse-generated from the library, granting the library raises population-search success from 40.5\% (de~novo@200) to 77.5\% (in-graph@200), statistically indistinguishable from EditGraph (McNemar $p{=}0.65$). At a matched 29-query budget on the shared graph, EditGraph retains a large lead over In-graph GA (79.5\% vs.\ 50.0\%). Matched-budget de~novo GraphGA collapses to 7.0\%. These experiments replace prose-only fairness disclaimers with controlled ablations of search space and budget.

\section{MMFF Protocol and Scheme~B Cost Accounting}
\label{app:mmff}

Scheme~B (script \texttt{36\_mmff\_scheme\_b.py}; audit dump \texttt{results/protocol\_audits/mmff\_scheme\_b\_accounting.json}) separates offline and online oracle spend.

\begin{itemize}
  \item \textbf{Energy oracle.} MMFF94 after ETKDGv3 embed (deterministic per-SMILES seed; single conformer; fallback \texttt{EmbedMolecule}); minimize $\le$200 iterations; failures unavailable; fixed SMILES$\to$$E$ cache; \emph{no} heavy-atom normalization.
  \item \textbf{$Q_{\mathrm{offline}}$.} $\approx$300 node-level $(q,\mathrm{LogP},E)$ labels on an edge-centric train-endpoint pool, acquired once before search, plus episode start/endpoint warmup (post-warmup cache size 330). These labels fit MMFF direct / effect / UCB. Effect targets are $\Delta p=p(b)-p(a)$ on co-labeled endpoints.
  \item \textbf{$Q_{\mathrm{online}}$.} Hard unique-node budget during beam search for molecules \emph{outside} the offline cache. Cache hits cost 0 for every method, including Random.
  \item \textbf{$Q_{\mathrm{total}}$ (learning).} $Q_{\mathrm{offline}}{+}Q_{\mathrm{online}}$ for fitted MMFF surrogates; Random/transfer match on free cache access but not on MMFF training spend. Reported hard budgets $Q{\in}\{10,20,40\}$ are online-only.
  \item \textbf{Overlap.} Approximated 300-pool SMILES have zero start/endpoint/path overlap with the $n{=}30$ test episodes under the reconstruction used in the audit script; scaffold Jaccard is 0.
  \item \textbf{Statistics.} With $n{=}30$ the success step is $3.33$\,pp. McNemar exact tests vs.\ Random at $Q{=}40$ all give $p{\ge}0.13$ (\texttt{results/mmff\_stats/mmff\_statistical\_checks.json}). Transfer ($n{=}50$): Random vs.\ Effect $p{=}0.58$ (Suite~A reversal \emph{not} confirmed); Random vs.\ Direct $p{=}0.021$. Main-text claim: \emph{no detectable MMFF-surrogate advantage} / not a confirmed Random$\succ$MLP reversal.
\end{itemize}

\section{Information Sufficiency (Extended Remarks)}
\label{app:theory}

Proposition~\ref{prop:equiv} in the main text is a representational / reparameterization observation: under deterministic $p$ and complete visible structure $x(m')$, ideal direct and effect maps recover the same $p(m')$ and therefore the same navigation scores $S(m')=d(\widehat p(m'),p^\star)$. It separates $L_{\mathrm{pred}}(\widehat p,p(m'))$ from $S$; it does not claim Bayesian decision equivalence under a prior. Empirical models use $\phi_{\mathrm{Morgan}}$, which is not assumed sufficient.

If instead $p(m)=\bar p(m)+\xi$ with assay noise $\xi$, or if conformers are sampled stochastically at query time, planner-visible $x(m')$ may not determine the realized label and equivalence can fail. Our MMFF protocol uses a fixed ETKDG seed and a per-SMILES cache, so $E_{\mathrm{MMFF}}$ remains a deterministic function of SMILES under that protocol; MMFF results therefore probe transfer and finite-data surrogates rather than a counterexample to Proposition~\ref{prop:equiv}.
